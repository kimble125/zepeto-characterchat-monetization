# AI 자기소개서·포트폴리오 작성용 Handoff

이 문서는 Claude, GPT 등에게 이 프로젝트를 설명하고 자기소개서나 포트폴리오 문구를 작성하게 할 때 함께 제공하기 위한 요약본입니다. 여기서 "학습"은 모델을 실제로 재학습시키는 것이 아니라, 대화 입력 맥락으로 제공한다는 뜻입니다.

## 프로젝트 기본 정보

- 프로젝트명: 제페토 캐릭터챗 수익화·UX 분석 대시보드
- GitHub URL: https://github.com/kimble125/zepeto-characterchat-monetization
- 공개 표시 제목 추천: `ZEPETO CharacterChat Monetization & UX Analytics`
- 핵심 질문: 제페토 캐릭터챗은 어떤 수익화 레버로 사용자 경험을 덜 해치면서 매출 가능성을 높일 수 있는가?
- 핵심 메시지: 공개 자료와 논문, 직접 사용 관찰을 기반으로 AI 캐릭터챗의 수익화 구조와 UX 리스크를 함께 분석하고, SQL·EDA·군집분석·광고 시뮬레이션·Streamlit 대시보드까지 end-to-end로 구현했다.

## 반드시 밝혀야 할 한계

이 프로젝트는 실제 제페토 내부 매출, ARPU, 이탈률, 광고 단가, 유저 로그를 사용하지 않았다. 공개 웹 자료, 직접 관찰한 가격·UX 메모, 공개 논문, 명시적 가정 기반 시뮬레이션으로 만든 공개 데이터 기반 분석이다. 따라서 실제 매출 예측이 아니라 수익화 전략 가설 생성과 실험 설계를 위한 포트폴리오 프로젝트로 설명해야 한다.

## 사용한 주요 자료

GitHub URL 외에 자기소개서·포트폴리오 작성 AI에게 함께 제공하면 좋은 자료는 다음이다.

1. `README.md`: 프로젝트 소개, 실행 방법, 핵심 산출물, KPI, 출처 요약
2. `docs/methodology.md`: 분석 방법론, 가정, 한계
3. `docs/reviewer_guide.md`: 검토자가 프로젝트를 빠르게 확인하는 방법
4. `screenshots/dashboard_overview.png`: 대시보드 첫 화면 스크린샷
5. `sql/analysis_queries.sql`: SQL 역량을 보여주는 분석 쿼리
6. `docs/erd.md`: ERD와 DB 설계
7. `src/data.py`: 출처 레지스트리, 가격 정제, SQLite 생성 로직
8. `src/clustering.py`: KMeans 군집분석 로직
9. `src/simulation.py`: 광고 시점 시뮬레이션 로직
10. 지원할 회사/직무의 채용 공고 원문
11. 본인의 이력서 또는 자기소개서 초안

로컬에 있는 원자료 PDF와 가격 관찰 markdown은 공개 repo에는 포함하지 않았다. 필요하면 자기소개서 작성 AI에게 별도 첨부해도 좋지만, 공개 제출물에는 GitHub URL과 대시보드 배포 URL을 우선 사용한다.

## 분석 파이프라인 요약

1. 분석 질문 정의: 사용자 경험을 해치지 않는 수익화 전략을 핵심 질문으로 설정
2. 데이터 수집: Character.AI, Talkie, Replika, ZEPETO, Zeta의 가격·플랜 구조와 공개 자료, 논문, 직접 사용 관찰값 수집
3. 데이터 정제: 가격, 결제 주기, 월 환산가, 재화 단위, 출처 유형, UX 피처 표준화
4. DB 구축: SQLite에 `services`, `pricing_plans`, `monetization_features`, `ux_research_factors`, `ad_observations`, `simulation_assumptions`, `kpi_results`, `sources` 테이블 구축
5. SQL/EDA: 최저 진입가, 플랜 수, 월 환산가, 수익화 복잡도, 재화 의존도, UX 지표 분석
6. 논문 기반 변수화: 외형/언어 스타일, 3D 캐릭터 AI, 실시간 입력 인식, 페르소나 일관성 연구를 KPI로 변환
7. 머신러닝: 표본이 작으므로 예측 회귀 대신 KMeans로 서비스 수익화 유형을 군집화
8. 광고 시뮬레이션: 직접 관찰한 4번째/19번째 발화 광고 시점을 기준으로 광고 수익 가능성과 UX 위험을 비교
9. 대시보드 구현: Streamlit과 Plotly로 분석 결과를 탭 구조로 시각화
10. 전략 가설 도출: 월드/아바타 보상형 광고, 캐릭터별 기억 패키지, 1일/3일 체험 패스 실험 제안

## 주요 KPI

- Entry Price Index: 최저 진입가 기준 접근성 점수
- Monetization Complexity Score: 구독, 재화, 광고, 단기 패스 등 수익화 구조의 복합성
- Metaverse Integration Score: 3D/월드/캐릭터 AI 결합 가능성
- Immersion Fit Score: 캐릭터 외형과 세계관이 채팅 몰입을 돕는 정도
- Persona/Language Fit: 캐릭터 말투, 페르소나, 응답 스타일의 일관성
- Ad Revenue Potential: 광고 시점별 상대적 수익 가능성
- UX Risk Index: 수익화가 사용자 몰입을 방해할 위험
- Balanced Opportunity Score: 수익화 레버, UX 강점, 리스크를 함께 본 우선순위 점수

## 도출한 전략 가설

1. 월드/아바타 보상형 광고: 광고를 단순 interrupt가 아니라 캐릭터 꾸미기, 월드 행동, 보상 경험과 연결한다.
2. 캐릭터별 프리미엄 슬롯/기억 패키지: 페르소나 일관성과 장기 대화 기억을 유료 가치로 전환한다.
3. 1일/3일 캐릭터챗 체험 패스: 월 구독보다 낮은 심리적 장벽으로 고몰입 세션을 수익화 funnel에 연결한다.

## 자기소개서에 쓸 수 있는 요약 문장

공개 자료와 직접 사용 관찰을 기반으로 AI 캐릭터 서비스 5종의 가격·수익화 구조를 수집·정제하고, SQLite/SQL 기반 EDA, 논문 기반 UX 변수화, KMeans 군집 분석, 광고 시점 시뮬레이션을 수행했습니다. 이를 Streamlit 대시보드로 구현해 제페토 캐릭터챗의 "사용자 경험을 해치지 않는 수익화" 가설을 도출했습니다.

## 더 자연스러운 포트폴리오 소개 문구

이 프로젝트는 AI 캐릭터챗 서비스의 가격·수익화 구조를 공개 자료 기반으로 비교하고, 사용자 몰입을 해치지 않는 수익화 전략을 탐색한 데이터 분석 대시보드입니다. 가격 정제, SQLite DB 구축, SQL 분석, 논문 기반 UX 변수화, KMeans 군집분석, 광고 시점 시뮬레이션을 하나의 Streamlit 대시보드로 연결했습니다.
