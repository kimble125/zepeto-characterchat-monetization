# Methodology

## 분석 질문

제페토 캐릭터챗은 어떤 수익화 레버로 사용자 경험을 덜 해치면서 매출 가능성을 높일 수 있는가?

## 데이터 범위

이 분석은 실제 제페토 내부 매출, ARPU, 이탈률, 유저 로그를 사용하지 않았습니다. 사용자가 제공한 로컬 문서, 직접 사용 관찰, 공식 웹 자료, 공개 논문, 산업 자료를 기반으로 만든 추정 분석입니다.

## 파이프라인

1. `source_registry` 작성: 로컬 문서, 공식 웹, 논문, 산업 자료를 출처 유형별로 통합했습니다.
2. 가격 정제: 구독, 단기 패스, 젬/코인/조각 패키지를 `pricing_plans`로 표준화했습니다.
3. 월 환산: 결제 주기가 다른 플랜을 비교하기 위해 `monthly_equivalent_krw`를 계산했습니다.
4. UX 변수화: 논문 근거를 `Immersion Fit`, `Persona/Language Fit`, `Metaverse Integration`, `Interaction Awareness`로 변환했습니다.
5. DB 구축: 정제 테이블을 SQLite에 적재하고 SQL 분석 쿼리를 별도 파일로 남겼습니다.
6. EDA/KPI: 진입가, 수익화 복잡도, 메타버스 결합도, UX 위험 지수를 계산했습니다.
7. KMeans: 표본이 작으므로 예측이 아니라 서비스 수익화 유형 분류 목적으로 사용했습니다.
8. 광고 시뮬레이션: 멱법칙 세션 길이 가정으로 광고 노출 시점별 수익 가능성과 UX 위험을 비교했습니다.

## 주요 한계

- 가격은 수집 시점, 국가, 앱스토어 정책에 따라 달라질 수 있습니다.
- 광고 노출 시점은 직접 사용 관찰값이므로 표본이 작습니다.
- CPM과 completion rate는 민감도 분석용 가정이며 실제 제페토 광고 단가가 아닙니다.
- KMeans는 서비스 5개 벤치마크에 대한 유형화 도구이며 일반화 목적의 모델이 아닙니다.

## Communication Notes

- 수익화 분석을 단순 가격 비교로 끝내지 않고 UX 논문 근거와 결합했습니다.
- SQL/SQLite를 통해 분석 재현성을 확보했습니다.
- 작은 데이터 문제에서는 회귀보다 군집 분석이 더 적합하다고 판단했습니다.
- 실제 내부 데이터가 없다는 한계를 숨기지 않고, 공개 데이터 기반 가설 생성 프로젝트로 포지셔닝했습니다.
